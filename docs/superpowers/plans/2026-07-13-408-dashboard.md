# 408 Study Console Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy a private, password-protected horizontal study console at `https://platform.arcol.site` for time windows, focus sessions, monthly focus heat, mock-exam metrics, and long-term study plans.

**Architecture:** A small Flask application serves the HTML shell and JSON endpoints. SQLite stores settings, focus sessions, scores, and weekly plans. The browser renders the desktop-first dashboard with vanilla JavaScript; the only visible navigation is the top status bar and a small settings entry. Authentication uses a single configured password, Flask's signed cookie session, `HttpOnly`, `SameSite=Lax`, and `Secure` cookies behind Nginx.

**Tech Stack:** Python 3.10+, Flask, Gunicorn, SQLite, pytest, vanilla HTML/CSS/JavaScript, systemd, Nginx, Certbot.
